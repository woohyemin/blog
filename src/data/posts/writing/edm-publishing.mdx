eDM 템플릿 퍼블리싱 작업을 하면서 겪었던 시행착오를 바탕으로 몇 가지 주의할 점들을 정리해 보았다.

# eDM 템플릿이란?

eDM(electronic Direct Mail)은 고객에게 발송하는 광고 및 안내 메일의 템플릿을 의미한다.

# eDM 템플릿 퍼블리싱할 때 주의할 점

eDM은 일반적으로 브라우저에서 보여지는 방식과 조금 달라서 의도한 대로 적용되지 않을 때가 많았다. 작업하면서 신경 썼던 부분들은 다음과 같다.

## <Emoji>1️⃣</Emoji> SVG 파일 사용을 지양한다.

Gmail에서 `SVG` 이미지가 정상적으로 표시되지 않는 이슈가 있었다. 따라서, eDM 템플릿에서는 **`svg` 대신 `png`나 `jpg` 포맷의 이미지를 사용**했다.

## <Emoji>2️⃣</Emoji> 레이아웃을 잡을 때 table을 사용한다.

일반적으로 웹 개발에서 레이아웃을 작업할 때 `div`를 자주 사용한다. 하지만 이메일 클라이언트에서 의도한 대로 레이아웃을 유지하려면 `div` 대신 `table`을 사용하는 것이 좋다. **다양한 이메일 클라이언트에서 일관된 레이아웃으로 보여지기 위해서 `table`을 사용**하여 레이아웃을 작업했다.

## <Emoji>3️⃣</Emoji> 스타일은 모두 인라인 스타일로 적용해야 한다.

모든 스타일은 각 태그 안에 인라인 스타일로 적용해야 한다. 하지만 인라인 스타일로 작업하다보니 **일부 스타일이 누락되지 않도록 더 신경 써야 했고, 디자인이 변경되는 경우에는 시간이 오래 걸리는 어려움**이 있었다.<Emoji>😰</Emoji>

이를 개선하기 위해 `class`로 적용된 스타일을 `inline-style`로 변환해주는 라이브러리를 찾아서 적용했고, 변경 사항에 빠르게 대응할 수 있게 되었다.

개발할 때는 기존에 작업하는 방식처럼 `class`로 스타일을 적용한 후, 빌드할 때 `inline-style`로 변환해주도록 하여 더 빠르고 편리하게 작업할 수 있었다.<Emoji>😀</Emoji>

## <Emoji>4️⃣</Emoji> 버튼 구현 시, button 태그를 사용하지 않고 버튼 이미지를 a 태그로 감싸는 방식으로 구현한다.

`button` 태그를 사용하거나 `a` 태그에 텍스트를 넣어 스타일링하면 의도한 대로 표시되지 않는 이슈가 있었다.

그래서 **버튼 이미지를 `a` 태그로 감싸서 구현하는 방식으로 작업**했다.
